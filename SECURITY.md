# Security Policy

## Supported versions

Security fixes are provided for the latest released version of Claude/Codex Sub Autocomplete.

## Report a vulnerability

Use [GitHub private vulnerability reporting](https://github.com/kkoemets/claude-codex-sub-autocomplete/security/advisories/new). Do not open a public issue for suspected credential exposure, unsafe provider invocation, prompt leakage, path disclosure, arbitrary file access, or command execution.

Include:

- the affected plugin and IntelliJ IDEA versions;
- the selected provider and authentication type, without credentials;
- minimal reproduction steps using non-sensitive sample code;
- the expected and observed impact; and
- any suggested mitigation.

Do not attach private repositories, prompts, source files, tokens, CLI configuration, or unredacted logs. You should receive an acknowledgement within five business days. A disclosure timeline will be coordinated after the report is reproduced and assessed.
